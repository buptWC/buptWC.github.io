Binary Search算法基本套路

本文默认读者是有一定算法基础的，对于基本的语法和bianry search概念都了解。本文从实际题目出发，尽可能让读者对需要使用二分法的题目更加敏感（能判断这道题需要使用二分），对二分法的代码写法更加了解。

基本二分法写法

就如同bfs详解中一样，基本的算法写法是最重要的一部分，因为决大部分题目的二分法代码都是在这个基础上进行一些改动即可。

对于一个有序数组A，需要在A中判断是否存在target

那么这里有几个点要注意一下：

  1. 命名方式。这里采用的lo,hi表示low, high，你也可以使用l,r表示left, right，这样有个什么好处呢？因为大家都这样命名，当你也有这个习惯的时候，你阅读别人的代码时一看到这两个变量名就可以立即反应过来这是二分！
  2. 计算mid的方式。可能已经有人注意了，为什么第5行里面写的不是mid = (lo+hi)//2，明明两种写法都是表达的一个意思。这是因为当lo,hi比较大的时候，lo+hi可能会溢出导致mid的计算结果不准确，当然在python里面这种情况是不存在的，但是c++,java里面都需要注意这点，所以在这里提一下。
  3. 指针的移动方式。在这道题中是lo=mid+1; hi=mid，那是因为当A[mid]<target之后，mid这个点本身一定不是正确答案，所以边界lo可以从mid的下一个开始取。那有人可能会问，那为什么hi=mid而不是hi=mid-1呢，明明mid也取不到。关键点来了，那是因为我们写二分法的时候，通常只有一个边界需要+-1，而另一个不需要，也就是要么是lo=mid+1;hi=mid，要么是lo=mid;hi=mid-1。这样写是肯定不会错的，因为他可以完整的遍历所有的值而不会出现任何漏掉的情况。（读者可自行检验）

那么了解这几点之后，我们在这里做一个小总结，同样以小见大，总结一下二分法的流程。

  1. 寻找左右边界值，也就是lo,hi的初值。
  2. 根据判定条件决定指针移动情况。
  3. 遍历过程中找到结果则返回 或 出循环后返回

现在我们来看上面那个例子的一个小变种：
对于一个有序数组A，需要在A中找到一个target，若找到则返回下标，若无则返回第一个比target大的值的下标。

我们使用刚才总结的流程来分析一下这道题

  1. 寻找左右边界值。显然应该是在整个数组中寻找，所以为0,n-1
  2. 决定指针移动情况。当A[mid]<target的时候，此时mid一定不可能是结果，可以跳过，那么hi可以不用分析了，直接写成hi=mid即可。那当A[mid]>target的时候，可以跳过mid吗？显然不行，因为题目要返回比target大的第一个值的下标，如果跳过的话可能会把答案排除了！
  3. 返回结果。若找到target，显然返回target。若没找到，出循环后lo==hi，任意返回一个即可。

OK，介绍到这里我认为基本写法应该也有个大概印象了，接下来的内容才是二分法的重点，那就是如何判断一道题是否该使用二分法？

先说结论，主要有以下几种题型：

  1. 多次问询题。给定一些数据，然后按照一定的规则进行多次问询，这种题型每次的回答所花费时间必须极小，通常为O(1)或O(logn)。
  2. 在极大的范围找寻某个值，这个值需要满足一定的条件。
  3. 题目数据量较大（无法使用n^2复杂度），但此时又需要多次遍历。

那么在介绍具体的例子之前，这里先稍微介绍一下python里面内置的关于二分搜索的模块bisect
例如对于A = [1,2,2,2,3,3,3,4]，如果我们要寻找插入target=3的位置的下标，有以下几种方法，看具体情况使用：

  1. bisect.bisect_left(A, target). 找到最左边的3的下标，这里会返回4
  2. bisect.bisect_right(A, target). 找到最右边的3的下标，这里会返回6

那么使用bisect模块去解决上一道题就非常简单了

Leetcode 981 Time Based Key-value Store

建立一个支持两种操作的类TimeMap，两种操作分别是
set(key, value, timestamp)方法，将key和value按照给定的timestamp存储
get(key, timestamp)方法，返回已经存储的key中pre_timestamp <= timestamp的值，如果有多个结果，返回pre_timestamp最大的那个值。如果没有结果则返回''.

示例：
输入: inputs = [“TimeMap”,”set”,”get”,”get”,”set”,”get”,”get”], inputs = [[],[“foo”,”bar”,1],[“foo”,1],[“foo”,3],[“foo”,”bar2”,4],[“foo”,4],[“foo”,5]]
输出: [null,null,”bar”,”bar”,null,”bar2”,”bar2”]

note:
对于每个测试样例，set() and get()函数会调用总共120000次。

这道题就属于典型的多次问询题，对于题目中所给的两个操作set()和get()，我们都必须以很小的时间复杂度去完成一次操作。

对于相同的key来说，里面的val会根据timestamp排序，那么我们只要在里面找到第一个小于等于询问的timestamp的值就好。

上一道在二分里面属于最简单的一类，很容易就能想到是二分，如何二分也很直接，就在列表里面找就行了。但是在更多的题里面却不是这样，看接下来这道题。

Leetcode 878 Nth Magical Number

一个数如果能被A或B整除，我们就认为这个数是’magical number’
现在需要返回第N个’magical number’，结果可能非常大，需要mod 10**9+7

示例 1:
输入：N = 1, A = 2, B = 3
输出：2
示例 2:
输入: N = 4, A = 2, B = 3
输出: 6

note: 1 <= N <= 10^9, 2 <= A,B <= 40000

由于N的范围过大，想通过遍历来一个个分析显然不行。其实这道题就是标准的在极大的范围内找一个数，这个数需要满足一定的条件
在这道题里这个条件是什么？就是有N-1个更小的’magical number’，对于某个数X，比X小的且是A的倍数的个数应该是X // A，那么比X小的且是B的倍数的个数应该是X // B，考虑到这些数中可能有重复计算的情况，那么既是A的倍数且是B的倍数的个数应该是X // M，其中M是A,B的最小公倍数.
